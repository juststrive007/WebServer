解决空请求的问题
HTTP协议中运行客户端发送空请求
相当于客户端TCP连接服务端口没有发送标准HTTP请求内容
，而是直接与服务端断开连接了
但是我们在接收请求后，ClientHandler会解析请求，
此时解析请求行读取到的就是空字符串，但是我们按着空格拆分三
项（method,uri,protocol）并进行相关的赋值操作
就会出现下标越界了
针对空请求，我们的处理应当是忽略本次操作。

思路：
当HttpRequest解析请求行遇到空请求时，主动对外抛出
一个空请求异常（自定义的异常），并最终通过构造方法
抛出给ClientHandler，当Clienthandler实例化HttpRequest
时捕获了空请求会直接跳到catch操作，这就等于忽略了后续
处理请求和响应客户端操作了
