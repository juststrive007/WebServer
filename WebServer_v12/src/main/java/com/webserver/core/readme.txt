此版本开始完成用户注册功能。

通过这个版本，我们要了解用户是如何将信息提交服务端的，以及服务端
是如何通过请求解析用户在页面输入的信息并进行处理。这个操作是在
request解析的时候进行的，因此这是一个通用操作，这个操作做好
后无论将来用户请求什么样的业务，传递数据时request都可以处理好
并交给处理业务的环节使用。

注册的流程：
1：用户请求页面注册
2：在注册页面中输入用户注册信息
3：点击注册按钮提交注册请求
4：服务端处理该请求
5：响应注册结果页面（成功或失败）

要点
1：理解页面中form表单的使用
2：GET请求提交数据的URL格式
3:服务端针对不同类型的请求的不同处理方式：
    3.1请求静态资源（已实现）
    3.2请求一个业务


实现：
1:在webapps/myweb 目录下新建注册页面：reg.html
2：在提交页面后，发现：url的抽象路径部分含有参数
  因此，我们在HTTPRequest解析请求行时，需要对抽象路径（uri的值）进行
  进一步的解析。
  2.1 在HttpRequest中添加三个新属性：
    String requestURI：存储请求部分（uri的？左侧内容）
    String queryString：存参数部分（uri的？右侧内容）
    Map parameters:存每一个参数

3. 在HttpRequest中添加一个方法：parseURI
    用于进一步解析uri的值，并在解析请求行的parseRequestLine中
    得到uri后调用来进一步解析。
