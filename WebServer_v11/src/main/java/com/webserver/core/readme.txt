继续重构代码
上一个版本中，我们已经实现了根据用户请求的实际资源响应对应的Content-Type值，
使得浏览器可以正确理解其请求的资源了（学子商城页面应当正常显示了）

但是有一个问题，HttpResponse的setEntity方法每次在设置正文同时
添加响应头Content-Type前都要创建一个Map保存所有资源后缀与对应头的值，
但是这个Map的内容实际是固定的，不用每次都创建，因此我们应当复用同一个
Map即可。
实现：
1.在http包中定义一个类：HttpContext
2.在HttpContext中定义静态属性Map mimeType
3.在静态块中初始化这个Map，将所有的资源后缀与对应得Content-Type值
存入其中
4.提供一个静态的公有方法可根据后缀获取对应得值
5.修改HTTPResponse的setEntity方法，在获取到资源后缀名后
调用该静态方法获取消息头的值并设置。

HTTP协议还有一些固定的内容，将来都定义在HttpContext中，比如我们可以将CR，LF在这里
定义，然后请求对象与响应对象都使用这个，而不是自己直接使用13,10这个编码。
