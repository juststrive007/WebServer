此版本在myweb/index.html页面上显示一张图片，，理解
浏览器显示页面的过程

由此得知，当页面上有其他素材时，浏览器会再次自动发起
请求，向服务端请求该素材，得到后将其进行显示。

从baidu.com 上下载商城的素材，并拷贝到我们的webspps
目录下，进行访问后发现页面无法正确显示。
查明原因：
浏览器请求的资源，我们通过打开循环接收客户端连接后
能成功将资源响应回去，但是，每次发送响应时，响应头
Content-Type都是发送固定值：text/html,使得浏览器
不能正确理解其请求的资源所导致。

因此，本版本开始，要解决发送响应头的问题：
1.现在HTTPResponse方法的sendHeaders发送响应
头是固定的只发送两个头：Content-Type与
Content-Length
而实际应用中，会结合实际情况发送很多头。
2.发送Content-Type时，值是固定的text/html


先解决第一个问题，将原有发送固定的头改成可以进行设置。
1：在HTTPResponse中定义一个Map，用于存放所有要发送客户端的响应头
2：对外提供一个put方法，让外界可以通过调用这个方法将要发送的响应头进行设置
3：修改sendHeaders方法，改为通过遍历Map将所有要发送的响应行进行发送
4：ClientHandler的处理请求环节，根据处理情况添加设置响应头的工作。
