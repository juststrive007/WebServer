此版本继续完成第三步：响应客户端的工作
上一个版本中已经完成了响应客户端请求的资源的
操作了，此版本完成如果请求的资源不存在，响应404
页面操作

实现
1：在webapps目录下创建一个字目录root
2：在root目录下新建一个页面404.html
    之所以将该页面放在root目录里，而不是放在myweb目录中是因为无论
    将来用户请求的是哪个网路应用中的资源时，都可能存在该
    资源不存在的情况，因此响应404的这个页面时一个通用页面，不应该
    是某个具体网络应用的独有页面。
3：完成ClientHandler处理请求的另一个分支，就是当资源不存在时，
    再次发送一个响应给客户端，其中：状态行中的状态码为404，描述内容
     为Not Found
     例如： HTTP/1.1 404 Not Found
     这里的Not Found之间有空格没有关系

     响应头参考上个版本响应正确资源
     响应文本则是将404.html页面内容响应。
