解决地址栏传输中文的问题

地址栏传输数据是在"?"右侧开始拼接内容（注册用户的操作中已经使用过）
但是对于URL中的抽象路径部分会被包含在请求的请求行当中传递,而HTTP
协议要求这部分出现的文本数据是不能出现非ISO8859字符集的字符（中
文不可以）
因此，这种情况的解决办法是：
浏览器会将中文这类字符安装UTF-8编码（浏览器自行指定，也可以通过JS
自行指定并转换）转换为一组字节，再将每个字节以2位16进制形式表示，前面
添加%，形成格式如%E8%82%83
因为上述格式中出现的字符时符合ISO8859规则的，因此可以进行传递，
而服务端则需要再反向操作将他们还原成对应得文字来达到传递中文的目的。

24  0001 1000
-24 1110 1000
